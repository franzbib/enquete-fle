# Modèle de scénario — Le dossier disparu

## 1. Objet du document

Ce document définit le premier scénario prototype du projet `enquete-fle`.

Il ne s’agit pas encore d’un scénario définitif entièrement dialogué, mais d’un modèle de travail destiné à guider Codex, Antigravity, l’agent scénariste et l’agent didactique.

Ce scénario doit permettre de tester :

- le moteur de scénario ;
- la navigation entre lieux ;
- la consultation de documents ;
- le système d’objets ;
- les énigmes de déduction ;
- le système d’indices ;
- l’accusation finale ;
- l’intégration discrète des objectifs FLE/FOU.

## 2. Titre provisoire

**Le dossier disparu**

## 3. Niveau

B1+ / B2 léger.

Le scénario doit rester accessible à des apprenants B1 solides, mais proposer assez de croisements d’informations pour intéresser des étudiants B2.

## 4. Durée visée

Durée cible : 40 à 45 minutes.

Version prototype minimale : 20 à 30 minutes.

## 5. Cadre général

L’enquête se déroule à l’ISPA, à Amiens, dans un environnement de préparation universitaire pour étudiants internationaux.

L’action se situe en fin d’après-midi, peu avant la fermeture du secrétariat.

Un dossier administratif important a disparu. Ce dossier devait être transmis rapidement pour une candidature universitaire.

Le joueur doit découvrir ce qui s’est passé.

## 6. Intention ludique

Le joueur doit avoir l’impression de mener une enquête documentaire.

Il doit :

- explorer des lieux ;
- consulter des documents ;
- lire des témoignages ;
- obtenir quelques objets ;
- repérer des contradictions ;
- reconstituer une chronologie ;
- formuler une accusation finale.

L’affaire ne doit pas être trop dramatique. Il ne s’agit pas d’un meurtre, mais d’un mystère administratif suffisamment important pour créer une tension narrative.

## 7. Intention pédagogique

Le scénario permet de travailler :

- la compréhension écrite ;
- la lecture de documents administratifs ;
- la reconstitution chronologique ;
- la comparaison de témoignages ;
- la formulation d’hypothèses ;
- la justification d’un raisonnement ;
- le passé composé ;
- l’imparfait ;
- le plus-que-parfait ;
- le conditionnel d’hypothèse ;
- les connecteurs temporels et logiques ;
- le lexique de l’université et de l’administration.

Ces objectifs ne doivent pas apparaître directement comme des exercices scolaires dans l’interface joueur.

## 8. Synopsis

Un dossier de candidature universitaire a disparu du secrétariat de l’ISPA.

Le dossier appartenait à une étudiante qui devait l’envoyer avant 18h00. Il contenait plusieurs documents importants : formulaire signé, attestation de niveau, justificatif d’identité, relevé de notes traduit et lettre de motivation.

Plusieurs personnes sont passées près du secrétariat entre 16h00 et 17h00.

Le joueur doit déterminer :

- qui a pris ou déplacé le dossier ;
- à quel moment ;
- pour quelle raison ;
- avec quelles preuves.

L’affaire doit rester nuancée : le responsable n’est pas nécessairement malveillant. Il peut avoir déplacé le dossier par erreur, pour aider quelqu’un ou pour dissimuler une autre négligence.

## 9. Lieux principaux

### 9.1 Secrétariat

Lieu central de l’affaire.

Contient :

- planning de passage ;
- mail imprimé ;
- badge visiteur ;
- premier témoignage.

Fonction :

- introduire l’enquête ;
- donner les premiers horaires ;
- fournir un objet d’accès.

### 9.2 Couloir

Lieu de transition.

Contient :

- note manuscrite ;
- témoignage d’un étudiant ;
- éventuelle fausse piste.

Fonction :

- faire apparaître une contradiction ;
- introduire un second suspect.

### 9.3 Salle informatique

Lieu verrouillé au départ.

Nécessite :

- badge visiteur ou code.

Contient :

- historique d’impression ;
- brouillon de mail ;
- indice numérique.

Fonction :

- fournir une preuve importante ;
- montrer qu’un document a été imprimé ou modifié.

### 9.4 Bibliothèque ou salle de travail

Contient :

- témoignage d’un étudiant ;
- ticket de bus ou note oubliée ;
- document secondaire.

Fonction :

- vérifier un alibi ;
- enrichir la chronologie.

### 9.5 Café voisin

Lieu optionnel dans la version enrichie.

Contient :

- ticket ;
- témoignage indirect ;
- horaire externe.

Fonction :

- confirmer ou contredire un déplacement.

Pour la version prototype, seuls trois lieux sont obligatoires :

- secrétariat ;
- couloir ;
- salle informatique.

## 10. Personnages

### 10.1 Chen

Rôle : étudiante concernée par le dossier.

Profil :

- sérieuse ;
- stressée ;
- très attentive à son inscription ;
- craint d’avoir oublié une pièce.

Fonction narrative :

- victime apparente de la disparition ;
- source d’informations partielles ;
- peut avoir elle-même déplacé un élément sans s’en souvenir.

Fonction linguistique :

- témoignage au passé composé et à l’imparfait ;
- expression de l’inquiétude ;
- marqueurs temporels.

### 10.2 Fahad

Rôle : étudiant passé au secrétariat pour demander une attestation.

Profil :

- pressé ;
- un peu confus ;
- affirme qu’il n’est resté que deux minutes.

Fonction narrative :

- suspect possible ;
- son emploi du temps contient une incohérence.

Fonction linguistique :

- chronologie ;
- contradiction entre durée déclarée et documents.

### 10.3 Xiaoyu

Rôle : témoin dans le couloir.

Profil :

- a vu plusieurs personnes passer ;
- ne connaît pas tous les détails ;
- rapporte parfois les paroles des autres.

Fonction narrative :

- témoin utile mais incomplet ;
- introduit le discours rapporté.

Fonction linguistique :

- discours indirect ;
- verbes de parole ;
- incertitude.

### 10.4 Madame Delorme

Rôle : secrétaire.

Profil :

- professionnelle ;
- affirme avoir laissé le dossier sur le bureau ;
- a été interrompue avant la fermeture.

Fonction narrative :

- donne le cadre administratif ;
- peut avoir commis une erreur involontaire.

Fonction linguistique :

- lexique administratif ;
- voix passive possible ;
- registre formel.

### 10.5 Monsieur Armand

Rôle : enseignant.

Profil :

- a récupéré des documents pour un cours ;
- dit ne pas avoir touché au dossier ;
- a utilisé la salle informatique.

Fonction narrative :

- suspect secondaire ;
- lien avec l’historique d’impression.

Fonction linguistique :

- justification ;
- conditionnel ;
- hypothèse.

## 11. Objets

Le nombre d’objets doit rester limité.

### 11.1 Badge visiteur

Type : objet d’accès.

Obtention :

- trouvé ou remis au secrétariat.

Fonction :

- accéder à la salle informatique.

### 11.2 Post-it avec code partiel

Type : objet d’accès / objet linguistique.

Obtention :

- trouvé dans le couloir ou sur le bureau.

Fonction :

- compléter un code à partir d’un indice textuel.

### 11.3 Ticket de bus

Type : objet de preuve.

Obtention :

- trouvé dans la bibliothèque ou près du café.

Fonction :

- vérifier l’horaire de déplacement d’un suspect.

### 11.4 Brouillon de mail

Type : objet de preuve / document.

Obtention :

- récupéré en salle informatique.

Fonction :

- prouver qu’un message a été préparé mais non envoyé.

### 11.5 Clé USB

Type : objet d’interaction ou preuve possible.

Obtention :

- version enrichie seulement.

Fonction :

- relier un personnage à l’impression d’un document.

Pour le prototype initial, utiliser seulement :

- badge visiteur ;
- ticket de bus ;
- brouillon de mail.

## 12. Documents

### 12.1 Planning du secrétariat

Type : planning.

Contenu :

- horaires de passage ;
- fermeture prévue ;
- rendez-vous de 16h00 à 17h00.

Fonction :

- base de la chronologie.

### 12.2 Mail de convocation

Type : mail.

Contenu :

- rappel de l’échéance ;
- heure limite ;
- documents attendus.

Fonction :

- expliquer l’enjeu.

### 12.3 Témoignage de Chen

Type : témoignage.

Fonction :

- fournir une version subjective des faits.

### 12.4 Témoignage de Fahad

Type : témoignage.

Fonction :

- créer une contradiction temporelle.

### 12.5 Note manuscrite

Type : note.

Fonction :

- donner un indice sur le déplacement du dossier.

### 12.6 Historique d’impression

Type : document technique simple.

Fonction :

- montrer qu’un document a été imprimé à une heure précise.

### 12.7 Brouillon de mail

Type : mail non envoyé.

Fonction :

- révéler qu’un personnage voulait prévenir quelqu’un mais ne l’a pas fait.

## 13. Énigmes principales

### 13.1 Énigme 1 — Reconstituer la chronologie

Type : ordering.

Objectif joueur :

Remettre dans l’ordre les événements entre 16h00 et 17h00.

Objectifs FLE :

- passé composé ;
- imparfait ;
- marqueurs temporels ;
- antériorité.

Documents nécessaires :

- planning ;
- témoignage de Chen ;
- témoignage de Fahad.

Résultat :

- débloque une nouvelle piste ;
- révèle une contradiction.

### 13.2 Énigme 2 — Identifier la contradiction

Type : contradiction.

Objectif joueur :

Comparer le témoignage de Fahad avec un document horaire.

Objectifs FLE :

- compréhension fine ;
- durée ;
- connecteurs temporels ;
- valeurs des temps.

Résultat :

- Fahad devient suspect plus sérieux ;
- accès à la salle informatique recommandé.

### 13.3 Énigme 3 — Utiliser un objet

Type : unlock.

Objectif joueur :

Utiliser le badge visiteur pour accéder à la salle informatique.

Objectifs FLE :

- comprendre la fonction d’un objet ;
- lire une consigne courte.

Résultat :

- accès à l’historique d’impression.

### 13.4 Énigme 4 — Relier impression et dossier

Type : matching.

Objectif joueur :

Associer l’historique d’impression, le brouillon de mail et un personnage.

Objectifs FLE :

- inférence ;
- justification ;
- cause / conséquence.

Résultat :

- débloque l’accusation finale.

### 13.5 Énigme finale — Accusation

Type : final-accusation.

Objectif joueur :

Choisir le responsable et trois preuves.

Résultat attendu :

Le joueur doit comprendre que le dossier n’a peut-être pas été volé, mais déplacé ou confondu avec d’autres documents.

## 14. Système d’indices

Chaque énigme doit proposer trois indices.

### Exemple pour l’énigme de chronologie

Indice 1 :

> Regardez d’abord les horaires explicitement indiqués.

Indice 2 :

> Quand une action était déjà terminée avant une autre action passée, le texte peut utiliser le plus-que-parfait.

Indice 3 :

> Le témoignage de Fahad ne correspond pas à l’heure indiquée dans le planning.

### Exemple pour l’énigme de contradiction

Indice 1 :

> Comparez la durée annoncée et les traces laissées dans les documents.

Indice 2 :

> Une personne qui dit être restée deux minutes ne devrait pas apparaître dans deux documents éloignés de quinze minutes.

Indice 3 :

> Fahad est encore lié au secrétariat après l’heure qu’il annonce.

## 15. Accusation finale

Le joueur doit sélectionner :

- un responsable ;
- au moins deux preuves ;
- une justification.

Réponses possibles :

### Résolution complète

Le joueur identifie le bon responsable et sélectionne les bonnes preuves.

### Résolution partielle

Le joueur identifie le bon responsable mais avec des preuves insuffisantes.

### Mauvaise piste

Le joueur accuse un personnage dont les preuves ne tiennent pas.

## 16. Solution provisoire

La solution exacte pourra évoluer.

Version recommandée pour le prototype :

Le dossier n’a pas été volé par malveillance.

Fahad a pris par erreur le dossier de Chen en pensant récupérer son attestation. Il s’est rendu compte de l’erreur plus tard, a voulu prévenir le secrétariat par mail, mais n’a pas envoyé le message. Le brouillon de mail et l’historique d’impression permettent de comprendre la confusion.

Cette solution présente plusieurs avantages :

- tension sans crime grave ;
- situation réaliste pour un contexte universitaire ;
- possibilité de fausses pistes ;
- rôle important des documents ;
- résolution compatible avec une classe FLE.

## 17. Chaîne des preuves

Preuve 1 :

Le planning montre que Fahad est passé au secrétariat.

Preuve 2 :

Son témoignage minimise la durée de son passage.

Preuve 3 :

L’historique d’impression ou un document montre qu’il est resté plus longtemps.

Preuve 4 :

Le brouillon de mail montre qu’il a voulu signaler une erreur.

Preuve 5 :

Le ticket ou un horaire externe confirme qu’il n’a pas pu revenir à temps.

Conclusion :

Fahad a pris ou déplacé le dossier par erreur, puis n’a pas immédiatement corrigé la situation.

## 18. Exploitation pédagogique possible

Après le jeu, l’enseignant peut proposer :

- reconstitution orale des faits ;
- rédaction d’un court rapport d’enquête ;
- travail sur les temps du passé ;
- transformation de témoignages au discours rapporté ;
- débat sur la responsabilité ;
- comparaison entre erreur, mensonge et dissimulation ;
- activité lexicale sur les documents administratifs.

## 19. Contraintes pour les agents

Les agents ne doivent pas :

- transformer l’affaire en meurtre ;
- ajouter trop de personnages ;
- rendre Fahad caricaturalement coupable ;
- créer une solution trop évidente ;
- ajouter des documents trop longs ;
- transformer les énigmes en exercices scolaires ;
- complexifier l’inventaire ;
- ajouter d’animation ;
- figer l’identité graphique.

## 20. Critère de réussite du scénario

Le scénario est réussi si le joueur peut dire :

> Au début, je pensais que le dossier avait été volé. En réalité, les documents montrent qu’il a été déplacé par erreur, puis que quelqu’un a essayé de cacher ou de corriger cette erreur trop tard.

Cette conclusion doit être obtenue par déduction, non par révélation arbitraire.
