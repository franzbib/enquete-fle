# Cahier des charges général — Jeu d’enquête FLE/FOU

## 1. Objet du document

Ce document définit le cahier des charges général du projet `enquete-fle`.

Il doit servir de référence aux agents IA, notamment :

- l’agent chef d’orchestre ;
- Codex ;
- Antigravity ;
- l’agent scénariste ;
- l’agent didactique ;
- l’agent testeur critique ;
- l’agent documentaliste.

Tout agent travaillant sur le projet doit respecter ce cahier des charges.

## 2. Définition du projet

Le projet consiste à créer un jeu d’enquête FLE/FOU en ligne, jouable sur navigateur, destiné à des apprenants de français de niveau B1/B2.

Le jeu doit être :

- en 2D ;
- statique ;
- sans animation ;
- structuré autour d’une enquête ;
- fondé sur la consultation de documents ;
- jouable en classe ou en autonomie guidée ;
- techniquement simple ;
- progressivement améliorable.

## 3. Nature du jeu

Le jeu doit être conçu comme un jeu d’enquête documentaire.

Le joueur explore des lieux, consulte des documents, observe des indices, obtient des objets, interroge des personnages, repère des contradictions et formule une accusation finale.

Le jeu n’est pas :

- un escape game pur ;
- un quiz grammatical ;
- une fiche d’exercices déguisée ;
- une fiction linéaire sans manipulation ;
- un jeu d’aventure complexe ;
- un RPG ;
- un jeu animé.

Le modèle recherché est une enquête courte, dense, claire et stimulante.

## 4. Public cible

Le jeu vise principalement :

- des apprenants FLE de niveau B1/B2 ;
- des étudiants internationaux ;
- des étudiants en préparation universitaire ;
- des groupes de deux à quatre apprenants ;
- des enseignants de FLE/FOU.

Le jeu doit rester suffisamment clair pour des apprenants B1, mais assez stimulant pour ne pas ennuyer des apprenants B2.

## 5. Durée des enquêtes

Chaque enquête doit viser une durée comprise entre 35 et 55 minutes.

Format recommandé :

- briefing : 5 minutes ;
- exploration initiale : 5 à 10 minutes ;
- énigmes et collecte d’indices : 25 à 35 minutes ;
- accusation finale : 5 à 10 minutes.

Le temps de jeu doit rester compatible avec une séance de classe.

## 6. Principe pédagogique central

La langue française doit être un outil de déduction.

Les objectifs linguistiques doivent être intégrés à la progression de l’enquête.

À éviter :

> Complétez les verbes au temps qui convient.

À privilégier :

> Reconstituez les faits à partir de trois témoignages contradictoires.

Dans le second cas, les temps verbaux deviennent nécessaires pour comprendre la chronologie.

## 7. Objectifs pédagogiques possibles

Selon les scénarios, le jeu peut travailler :

- compréhension écrite ;
- compréhension de témoignages ;
- repérage d’informations explicites ;
- repérage d’informations implicites ;
- reconstitution chronologique ;
- comparaison de documents ;
- formulation d’hypothèses ;
- justification d’une conclusion ;
- argumentation orale ;
- médiation en groupe ;
- prise de notes ;
- reformulation.

## 8. Points linguistiques possibles

Le jeu peut intégrer notamment :

- passé composé ;
- imparfait ;
- plus-que-parfait ;
- futur simple ;
- futur antérieur ;
- conditionnel présent ;
- conditionnel passé ;
- discours rapporté ;
- voix passive ;
- hypothèse ;
- cause ;
- conséquence ;
- opposition ;
- concession ;
- modalisation ;
- registres de langue ;
- lexique administratif ;
- lexique universitaire ;
- lexique de la ville ;
- lexique des relations sociales ;
- lexique de l’enquête.

Ces points ne doivent pas être affichés comme des exercices dans l’interface joueur.

## 9. Structure d’une enquête

Chaque enquête doit comporter :

- un briefing initial ;
- des lieux explorables ;
- des personnages ;
- des documents ;
- des objets ;
- des énigmes ;
- des indices gradués ;
- une accusation finale ;
- une solution argumentée.

## 10. Lieux

Les lieux doivent être représentés de manière statique.

Ils peuvent être affichés :

- sous forme de carte ;
- sous forme de liste ;
- sous forme de plan ;
- sous forme d’écran fixe.

Chaque lieu peut contenir :

- un document ;
- un objet ;
- un témoignage ;
- une énigme ;
- une fausse piste ;
- un accès verrouillé.

Les lieux doivent être peu nombreux dans une première enquête : trois à six lieux au maximum.

## 11. Personnages

Chaque personnage doit avoir :

- un nom ;
- un rôle ;
- un portrait provisoire ou définitif ;
- une relation avec l’affaire ;
- un témoignage ;
- un mobile possible ;
- un degré de fiabilité ;
- éventuellement un objet associé.

Les personnages doivent être crédibles. Ils ne doivent pas être de simples fonctions grammaticales.

## 12. Documents

Les documents sont au cœur du jeu.

Types possibles :

- mail ;
- SMS ;
- note manuscrite ;
- planning ;
- formulaire ;
- règlement ;
- article ;
- rapport ;
- ticket ;
- reçu ;
- témoignage ;
- capture fictive ;
- extrait de conversation ;
- document universitaire ;
- document administratif.

Chaque document doit servir au moins l’une des fonctions suivantes :

- fournir une preuve ;
- créer une contradiction ;
- ouvrir une piste ;
- donner une fausse piste ;
- débloquer un objet ;
- éclairer un personnage ;
- permettre une déduction.

Aucun document long ne doit être ajouté sans nécessité ludique.

## 13. Objets

Le jeu doit inclure un nombre restreint d’objets.

Nombre recommandé par enquête :

- minimum : 3 ;
- maximum : 6 ;
- idéal : 4.

Les objets peuvent être :

- objets d’accès ;
- objets de preuve ;
- objets d’interaction ;
- objets linguistiques.

Un objet peut servir à :

- débloquer un lieu ;
- ouvrir un document ;
- convaincre un personnage ;
- vérifier une hypothèse ;
- établir une preuve ;
- résoudre une énigme ;
- soutenir l’accusation finale.

Les objets doivent être utiles. Un objet purement décoratif est à éviter.

## 14. Obtention des objets

Les objets peuvent être obtenus :

- par exploration ;
- par résolution d’énigme ;
- par dialogue ;
- par combinaison de documents ;
- par choix stratégique ;
- par consultation attentive d’un lieu.

Aucun objet indispensable ne doit être caché de manière injuste.

Si un objet est nécessaire à la progression, le joueur doit pouvoir comprendre :

- qu’il existe ;
- où il peut être cherché ;
- comment il peut être obtenu ;
- à quoi il peut servir.

## 15. Inventaire

L’inventaire doit rester simple.

Chaque objet doit afficher :

- son nom ;
- son icône ;
- sa description ;
- son lieu d’obtention ;
- son statut.

Statuts possibles :

- non trouvé ;
- trouvé ;
- utilisé ;
- preuve possible ;
- preuve utilisée.

L’inventaire ne doit pas devenir un système de type RPG.

## 16. Énigmes

Les énigmes doivent être liées à l’histoire.

Types d’énigmes recommandés :

- reconstitution chronologique ;
- comparaison de témoignages ;
- repérage de contradiction ;
- association document / suspect ;
- utilisation d’objet ;
- analyse de registre ;
- reformulation ;
- déduction finale.

Les énigmes ne doivent pas être arbitraires.

À éviter :

- code sans justification narrative ;
- énigme mathématique gratuite ;
- cadenas artificiel ;
- devinette opaque ;
- réponse impossible à inférer.

## 17. Indices

Chaque énigme importante doit proposer trois niveaux d’indice.

### Niveau 1 — Orientation

L’indice attire l’attention sur une zone utile.

### Niveau 2 — Raisonnement

L’indice aide à comprendre la logique.

### Niveau 3 — Déblocage

L’indice donne presque explicitement la solution.

L’usage des indices peut réduire le score, mais ne doit jamais humilier le joueur.

## 18. Accusation finale

L’accusation finale doit demander au joueur de :

- choisir un responsable ;
- sélectionner des preuves ;
- éventuellement choisir ou rédiger une justification ;
- vérifier la cohérence de son hypothèse.

Le jeu peut prévoir une réussite partielle.

Exemples :

- bon coupable, mauvaises preuves ;
- bonnes preuves, mauvaise interprétation ;
- accusation correcte et bien justifiée.

## 19. Score

Le score doit rester discret.

Il peut tenir compte :

- des énigmes résolues ;
- des objets trouvés ;
- des preuves correctement sélectionnées ;
- du nombre d’indices utilisés ;
- de la qualité de la résolution finale.

Le score ne doit pas dominer l’expérience.

## 20. Mode enseignant

À terme, le jeu doit prévoir un mode enseignant.

Ce mode doit fournir :

- niveau CECRL ;
- objectifs pédagogiques ;
- points linguistiques travaillés ;
- durée estimée ;
- déroulement conseillé ;
- corrigé ;
- solution finale ;
- pistes de prolongement ;
- grille d’observation.

Ce mode n’est pas prioritaire dans la V0.1, mais l’architecture doit permettre son ajout.

## 21. Contraintes techniques

La première version doit respecter les contraintes suivantes :

- React ;
- Vite ;
- TypeScript ;
- Tailwind CSS ;
- pas de backend ;
- pas de base de données ;
- pas d’animation ;
- pas de moteur de jeu lourd ;
- sauvegarde locale éventuelle ;
- scénarios séparés du moteur.

Les scénarios doivent être définis dans des fichiers de données.

Le moteur doit pouvoir charger plusieurs scénarios à terme.

## 22. Contraintes graphiques

L’espace graphique doit être prévu dès le départ.

Il faut prévoir :

- zone de carte ou de lieux ;
- zone de documents ;
- zone de suspects ;
- zone d’inventaire ;
- zone d’indices ;
- zone de carnet ;
- zone de résolution finale.

Cependant, l’identité graphique finale viendra plus tard.

Dans les premières versions, le graphisme doit être provisoire, sobre et fonctionnel.

## 23. Première enquête prototype

La première enquête prototype est :

**Le dossier disparu**

Cadre :

- ISPA ;
- Amiens ;
- environnement universitaire ;
- secrétariat ;
- salle informatique ;
- couloir ;
- bibliothèque ;
- café voisin.

Intrigue :

Un dossier important a disparu avant une échéance universitaire. Plusieurs personnages avaient un mobile ou une occasion. Le joueur doit reconstituer les faits.

Objectifs :

- vérifier le concept ;
- tester le moteur ;
- tester les objets ;
- tester les documents ;
- tester l’accusation finale ;
- vérifier l’intégration des objectifs FLE.

## 24. Logique d’amélioration progressive

Le projet doit avancer par étapes.

Versions prévues :

- V0.0 : cadrage documentaire ;
- V0.1 : squelette technique ;
- V0.2 : moteur minimal de scénario ;
- V0.3 : première enquête jouable ;
- V0.4 : inventaire et objets ;
- V0.5 : système d’indices ;
- V0.6 : amélioration UX ;
- V0.7 : mode enseignant ;
- V0.8 : espace graphique enrichi ;
- V0.9 : deuxième scénario ;
- V1.0 : première version publiable.

Chaque version doit être stable, testable et documentée.

## 25. Rôle de Codex

Codex doit intervenir principalement sur :

- architecture ;
- code ;
- moteur de scénario ;
- types TypeScript ;
- inventaire ;
- validation des énigmes ;
- tests ;
- documentation technique ;
- correction de bugs.

Codex ne doit pas :

- ajouter d’animations ;
- ajouter de backend non demandé ;
- complexifier inutilement ;
- modifier la vision pédagogique ;
- produire des énigmes arbitraires.

## 26. Rôle d’Antigravity

Antigravity doit intervenir principalement sur :

- interface ;
- UX ;
- espace graphique ;
- organisation visuelle ;
- lisibilité ;
- tests de parcours ;
- identité graphique progressive.

Antigravity ne doit pas :

- figer trop tôt l’identité finale ;
- casser le moteur ;
- ajouter d’animations ;
- surcharger l’interface ;
- privilégier le graphisme au détriment du jeu.

## 27. Critères de réussite

Le projet sera réussi si :

- le joueur mène réellement une enquête ;
- la langue sert la déduction ;
- les objectifs pédagogiques restent intégrés ;
- les objets sont utiles ;
- les énigmes sont justes ;
- l’interface est claire ;
- le jeu est utilisable en classe ;
- le code est maintenable ;
- une deuxième enquête peut être ajoutée sans tout refaire.

## 28. Ce qu’il faut absolument éviter

À éviter absolument :

- quiz grammatical ;
- exercices scolaires déguisés ;
- objets décoratifs ;
- interface confuse ;
- graphisme trop précoce ;
- animations ;
- backend prématuré ;
- énigmes arbitraires ;
- fausses pistes injustes ;
- documents trop longs ;
- inventaire trop complexe ;
- scénarios multiples avant stabilisation du premier prototype.

## 29. Principe final

Chaque élément du jeu doit servir :

1. l’enquête ;
2. l’immersion ;
3. l’apprentissage.

Dans cet ordre apparent.

Pour le joueur, le jeu doit dire :

> Tu mènes une enquête.

Pour l’enseignant, le jeu doit permettre :

> Tu fais travailler la compréhension, la grammaire, l’argumentation, la médiation et la justification.
