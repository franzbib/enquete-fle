# Résolution finale V0.6

## Objectif

La V0.6 ajoute une conclusion jouable au scénario `Le dossier disparu`.

Cette conclusion n'est pas une accusation violente. Elle demande au joueur de formuler l'explication la plus plausible à partir des pièces réunies.

Objectif de ton :

- expliquer plutôt que condamner ;
- rester prudent ;
- éviter l'humiliation d'un étudiant ;
- terminer sur une réparation administrative.

## Logique joueur

La résolution finale devient accessible après validation des deux énigmes principales :

- `chronologie-initiale` ;
- `contradiction-fahad`.

Le joueur doit ensuite :

1. choisir une hypothèse finale ;
2. sélectionner trois pièces justificatives ;
3. valider l'explication.

## Hypothèse correcte

Hypothèse attendue :

```text
Fahad a probablement récupéré ou gardé par erreur certains documents, puis a hésité à signaler la confusion.
```

Cette formulation reste volontairement modérée. Elle ne dit pas que Fahad a volé le dossier et ne le transforme pas en coupable public.

## Pièces justificatives attendues

Pièces principales :

- `temoignage-fahad` ;
- `historique-impression` ;
- `brouillon-mail`.

Rôle de ces pièces :

- le témoignage de Fahad donne sa chronologie déclarée ;
- l'historique d'impression montre une session à son nom à 16h48 ;
- le brouillon de mail suggère une confusion de documents.

Pièces secondaires présentes dans la liste :

- `temoignage-xiaoyu` ;
- `note-manuscrite` ;
- `planning-secretariat`.

Pièce décorative volontairement non décisive :

- `affiche-inscriptions-examens`.

Les objets d'inventaire, comme la clé USB `Exercices B1`, ne sont pas utilisés dans cette résolution finale.

## Feedbacks

Si l'hypothèse choisie va trop loin, le feedback rappelle que les preuves ne permettent pas d'affirmer cela.

Si les pièces sélectionnées ne soutiennent pas assez l'explication, le feedback invite à relier :

- la déclaration de Fahad ;
- la trace technique ;
- la confusion signalée.

Si la réponse est cohérente, le feedback insiste sur :

- l'absence de preuve de vol ;
- la confusion de documents ;
- la chronologie problématique ;
- le brouillon de mail comme signe d'une erreur remarquée.

## Fin heureuse

La conclusion indique que Delphine reprend le dossier avec Chen, vérifie les documents manquants, réimprime l'attestation nécessaire et remet le dossier en ordre.

Le dossier de Chen est finalement validé.

Fahad n'est pas publiquement accusé. Il comprend qu'il aurait dû signaler la confusion plus vite.

L'affaire se termine sans sanction, avec une règle simple : quand un document ne semble pas être le bon, il faut le dire immédiatement.

## Choix pédagogiques implicites

La résolution finale soutient discrètement :

- la formulation d'hypothèse ;
- la justification par preuves ;
- la prudence argumentative ;
- la distinction entre preuve et interprétation ;
- le lexique administratif ;
- la reformulation au conditionnel ou avec des marques de prudence.

Ces objectifs ne doivent pas apparaître dans l'interface joueur comme un exercice scolaire.

## Limites volontaires

La V0.6 ne crée pas :

- score complet ;
- sanction ;
- accusation brutale ;
- backend ;
- base de données ;
- deuxième scénario ;
- mode enseignant ;
- système complexe de preuves.

La résolution reste locale, simple et intégrée au scénario existant.
